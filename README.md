# Manual Testing Project

## Project description

This project presents manual testing of the purchase process in an e-commerce store.
The goal is to verify the correctness of the main shopping flow and to identify functional,
integration, and usability issues.

## Tested application

- Platform: PrestaShop (demo environment)
- Scope: login, product catalog, cart, checkout, payment
- Environment: demo / test environment

## Scope of testing

- User authentication (login / logout)
- Adding products to cart
- Cart management
- Checkout process
- Payment process (test payment) - Due to demo environment limitations, payment success flow could not be tested
- Basic UX evaluation

## Types of tests

- Manual functional tests
- Smoke tests
- Happy path tests
- Negative tests
- UX checks

## Tools

- Browser (Firefox)
- Manual test cases (Markdown)
- Bug reports (Markdown)

## Limitations

- Payment success flow could not be tested due to lack of available payment methods in the demo environment
- Tests were conducted on a demo instance with limited configuration options
